# UR Card 申请全攻略：Web3 原生瑞士 IBAN 万事达卡从零到一

UR 是由 SR Saphirstein AG 运营的瑞士持牌金融科技应用，基于 Mantle Network（以太坊 L2）构建，将**自托管加密钱包 + 瑞士 IBAN 银行账户 + 万事达借记卡**三合一，是目前最具 Web3 原生特色的 U 卡之一。**0 开卡费、闲置资金可生息、消费零损耗**是其最核心的三大差异化优势。

## 一、UR Card 核心优势

| 特性 | 说明 |
|------|------|

| **链上基础设施** | 基于 Mantle Network（ETH L2），Gas 费极低，链上透明 |
| **钱包类型** | MPC 非托管钱包，**无需助记词**，Google/邮箱账号即可登录 |
| **开卡 / 月费** | **完全免费**，0 开卡费、0 月费 |
| **充值费率** | Pro 会员（活动期）**0%**；普通用户约 **0.5~1%** |

## 二、UR 与其他 U 卡最大的差异

UR 的核心设计逻辑与 Fiat24 系（SafePal Card / Bitget Wallet Card）完全不同：

| 对比维度 | UR Card | Fiat24 系（SafePal / Bitget Wallet）|
|---------|---------|--------------------------------------|
| **底层账户** | 瑞士 IBAN（SR Saphirstein AG） | 瑞士 Fiat24 独立 IBAN |
| **钱包类型** | MPC 非托管，**无助记词**，Google 登录 | 需要助记词，完整私钥钱包 |
| **链上基础** | Mantle Network（ETH L2） | 以太坊 Arbitrum 链 |
| **闲置生息** | ✅ USDe 存入自动生息 3~5% | ❌ 不支持 |
| **消费逻辑** | 智能合约实时链上兑换 | 法币充值后消费 |
| **KYC 身份证** | 部分区域不支持，**护照更稳定** | 中国大陆身份证可用 |
| **开卡费** | **$0** | $0（邀请码后）|

**结论**：UR 更适合持有加密资产、希望闲置资金同时生息、又能随时刷卡消费的 Web3 用户；而 Fiat24 系更适合纯出金需求、绑定国内微信/支付宝的用户。

## 三、开卡前准备清单

- **护照**（推荐，稳定性最高；部分区域身份证不被接受）

- **科学上网工具**：境外 IP 环境，建议挂欧洲/美国/台湾节点
- **海外区 App Store / Google Play 账号**：下载 UR App
- **USDC 或 USDe**（Arbitrum / Base / Mantle 链）：用于激活和充值
- **首次消费至少 $5**：触发新用户 $5 奖励

## 四、注册步骤详解

### Step 1：下载 UR App

前往 App Store（海外区 Apple ID）或 Google Play 搜索「**UR: Fiat & Crypto Unified**」，或直接访问官网 **get.ur.app** 网页端注册。

### Step 2：创建账户

1. 打开 App → 选择「**使用 Google 登录**」或「邮箱注册」
2. 填写邀请码（如 **efk27b**，完成首消费 ≥$5 后奖励 $5）
3. 系统自动生成 **MPC 自托管钱包**，无需手动备份助记词

> ⚠️ MPC 钱包基于 Google 账号或邮箱绑定，账号安全即钱包安全。**务必开启 Google 账号二步验证（2FA）**，绑定手机号 + 备份码，防止账号被盗。

### Step 3：KYC 身份认证

进入 App → 「**Profile**」→「**Identity Verification**」

1. 选择证件类型：优先选「**Passport（护照）**」
2. 上传护照首页高清照片（正面信息页）
3. 完成**活体人脸识别**（按提示转头、眨眼）
4. 系统通常 **10~30 分钟**内审核通过，复杂情况最长 1~2 个工作日

> ⚠️ 中国大陆身份证在部分地区 / 版本中无法通过验证，强烈建议直接使用**护照**，避免反复 KYC 失败。

### Step 4：充值激活借记卡

KYC 通过后，进入「**Wallet**」→「**Deposit**」：

1. 选择充值币种：**USDC** 或 **USDe**
2. 选择充值链路：
   - **Arbitrum**（推荐，手续费低）
   - **Base** 或 **Mantle**（手续费极低）
3. 将交易所 USDC 提币至对应 UR 充值地址
4. 到账后资产会出现在 UR 钱包余额中

> 充值后无需额外操作，刷卡消费时智能合约会**自动实时将链上 USDC → 法币**完成支付，全程无感。

### Step 5：激活借记卡

进入「**Card**」→「**Activate Card**」：

1. 填写账单地址（可填瑞士真实格式地址，或系统自动分配）
2. 确认开启「**Card Enabled**」
3. 激活成功后即可查看**完整卡号、有效期、CVV**
4. 进行 **首笔 ≥$5 消费**，$5 奖励自动到账

### Step 6：激活闲置生息（进阶功能）

UR 的差异化核心功能：将卡内 **USDe** 通过 Ethena 协议自动生息：

1. 充值时选择 **USDe**（而非 USDC）
2. 余额自动参与链上质押，年化约 **3~5%**
3. 消费时智能合约自动从余额扣除，生息与消费同步进行，完全无感

> 这意味着你的「银行卡余额」不是闲置的，而是一直在 DeFi 协议里赚钱，直到你真正消费。

## 五、绑定支付宝 / 微信 / Google Pay / Apple Pay

**Google Pay / Apple Pay（推荐，最稳定）：**
1. Google 钱包或 Apple 钱包 → 添加付款方式 → 输入 UR 卡号、有效期、CVV
2. 完成银行短信验证绑定，可用于 NFC 支付及在线结算

**支付宝（AlipayHK 或国内大陆版）：**
1. 支付宝 → 银行卡 → 添加银行卡 → 输入 UR 卡号信息
2. 大陆版支付宝绑定成功率因用户不同有差异，**建议优先绑 AlipayHK**

**微信支付：**
> ⚠️ 用户反馈瑞士 IBAN Mastercard 卡段绑定微信**成功率不稳定**，部分用户成功，部分触发风控，建议不要过度依赖微信，优先选 Apple Pay / Google Pay。

## 八、适合人群

- **Web3 重度用户**：日常持有 USDC/USDe，希望闲置资金自动生息同时保留消费能力
- **无助记词需求用户**：担心助记词管理的新手，Google 账号登录更直觉
- **数字游民 / 跨境工作者**：需要瑞士 IBAN 账户收取国际汇款（自由职业 / 创作者收款）
- **追求极低消费损耗**：链上实时汇率兑换，避免法币充值的双重兑换损耗
- **不依赖微信支付的用户**：日常消费以 Apple Pay / Google Pay / 海外 App 为主

## 九、常见踩坑汇总

- **身份证 KYC 失败**：中国大陆身份证兼容性差，直接用护照申请，一次通过率大幅提升
- **Google 账号安全是关键**：MPC 钱包绑定 Google 账号，账号被盗即钱包被盗，**必须开启 Google 2FA 并妥善保管备份码**
- **链选错资产丢失**：充值须选 Arbitrum / Base / Mantle 链，不支持 TRC20（TRON）链，链选错资产几乎无法找回
- **USDe 和 USDC 充值区别**：只有充值 **USDe** 才能自动生息；充 USDC 不会产生利息收益
- **微信绑卡风控**：瑞士 IBAN 卡段绑微信成功率不稳定，不要指望微信作为主力支付，优先用 Apple Pay / Google Pay
- **Pro 会员才能享受 0% 充值费**：免费账户充值会产生费率，注册时留意活动期免费 3 个月 Pro 资格，活动结束后评估是否续费
- **实体卡仍在内测**：目前 UR 主要提供虚拟卡，实体卡功能尚未全面开放，需关注官方公告
- **消费小额也扣兑换费**：虽然损耗接近 0，但非美元区消费仍有 FX 费，小额高频消费累积起来有一定损耗，适合中大额跨境消费
- **DeFi 协议风险**：USDe 生息依赖 Ethena 协议，属于 DeFi 风险，极端市场行情下收益率会波动，本金安全性依赖协议稳定性，需自行评估
